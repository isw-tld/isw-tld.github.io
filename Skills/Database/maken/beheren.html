Gegevens beheren

Nu er een plan ligt voor de database en de tabellen klaar staan, kunnen we gaan beginnen met gegevens verwerken. Bij gegevens is het vaak van belang dat er vier type operaties gedaan kunnen worden, dit heten de CRUD operaties dit staat voor Create, Read, Update en Delete. Read hebben we natuurlijk al gezien met de select query's, en create hebben we bij het maken van de tabellen gezien. Hier gaan we nu leren invoegen, wijzigen en verwijderen van gegevens.

Toevoegen

Om gegevens te hebben in de database heb je wel gegevens nodig. De eerste stap als je een nieuwe database hebt is om de tabellen te vullen met data. Dit kan je doen met een query: 


  INSERT INTO <tabel> (<kolommen>) VALUES (<waardes per kolom>)
      

  INSERT INTO Tijdschriften VALUES
  ('Flammen', 'Toonder', 5.12),
  ('Druppels', 'Toonder', 4.23),
  ('Broeders', 'Nortwick', 5.39)

De kolommen opschrijven is optioneel, zoals je ziet in het voorbeeld hierboven. Als je dit niet doet dan betekent het dus dat je alle kolommen gaat invullen, en moet je dus ook een waarde voor alle kolommen geven. Als je bijvoorbeeld dan een tussenvoegsel wilt maken die leeg is gebruik je of twee quotes die je niet vult (''), of je gebruikt NULL als waarde. NULL kan alleen als de waarde in de kolom ook daadwerkelijk NULL mag zijn.

Hierboven heb je de basis gelezen, maar ik raad je persoonlijk aan om het een tikje aan te passen. als je INSERT IGNORE INTO <tabel> (<kolommen>) VALUES (<waardes per kolom>) gebruikt, dan geeft je database minder vaak errors als je later dingen toevoegt. De scheelt veel hoofdpijn...

Wijzigen

Voor gegevens wijzigen gebruiken we een UPDATE query. De UPDATE query heeft de volgende opbouw: 


  UPDATE <tabel>
  SET <kolom> = <waarde>
  WHERE <kolom> = <waarde>

Je kan verschillende dingen tegelijk updaten als je bij SET een komma gebruikt en daarna meer kolommen veranderd. De WHERE werkt hetzelfde als bij een gewone query. 


  UPDATE abonnees
  SET huisnummer = 234,
   Postcode = '2545JK'
  WHERE abonneeID = 12

Verwijderen

Als gegevens dan oud zijn, of een persoon beroept zich op het recht om vergeten te worden, dan heb je de DELETE query nodig. De DELETE query is als volgt opgebouwd: 


  DELETE FROM <tabel>
  WHERE <kolom> = <waarde>
      

  DELETE FROM abonnees
  WHERE abonneeID = 12

Ook hier werkt WHERE weer zoals je gewend bent.
